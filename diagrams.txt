## User
* id: Int @id @default(autoincrement())
* nom: String
* prenom: String
* email: String @unique
* motDePasse: String
* telephone: String?
* role: Role @default(CLIENT)  // CLIENT, SERVEUR, GERANT
* adresse: String?
* matricule: String?
* createdAt: DateTime @default(now())
* updatedAt: DateTime @updatedAt

Relations:
* clientOrders: Order[] @relation("ClientOrders")
* serveurOrders: Order[] @relation("ServeurOrders")
* reservations: Reservation[]
* reviews: Review[]

---

## Category
* id: Int @id @default(autoincrement())
* nom: String @unique
* products: Product[]

---

## Product
* id: Int @id @default(autoincrement())
* nom: String

* prix: Float
* disponibilite: Boolean @default(true)

## Order
* id: Int @id @default(autoincrement())

* client: User @relation("ClientOrders", fields: [clientId], references: [id])
* serveur: User? @relation("ServeurOrders", fields: [serveurId], references: [id])

3. ARCHITECTURE TECHNIQUE IMPLÉMENTÉE
----------------------------------------------------------

### Backend (Fastify + TypeScript + Prisma + PostgreSQL)
* Framework: Fastify v5
* ORM: Prisma Client v7 avec adapter PostgreSQL (@prisma/adapter-pg)
* Auth: JWT (@fastify/jwt) + bcryptjs
* Validation: Zod
* CORS: @fastify/cors (configuré pour web et mobile)
* Structure: Routes modulaires par domaine

### Frontend (React Native + Expo Router + TypeScript)
* Framework: Expo SDK 56 / React Native 0.85
* Navigation: Expo Router v3 (file-based routing)
* State: React Context (AuthContext, CartContext)
* API: Fetch natif avec token Bearer
* Icons: @expo/vector-icons (MaterialIcons)
* Safe Area: react-native-safe-area-context
* Storage: @react-native-async-storage/async-storage

### Structure de navigation (Expo Router)
```
app/
├── _layout.tsx              # Root + SafeAreaProvider + FAB panier
├── index.tsx                # Redirection vers /menu
├── menu.tsx                 # Menu public (catégories, recherche, FAB)
├── product/[id].tsx         # Détail produit + avis
├── (auth)/
│   ├── login.tsx            # Connexion + redirection par rôle
│   └── register.tsx         # Inscription (CLIENT uniquement)
├── (client)/
│   ├── _layout.tsx          # Guards + tabs (Panier, Commandes, Réservations, Profil)
│   ├── cart.tsx             # Panier avec +/- quantité, supprimer, commander
│   ├── orders.tsx           # Historique commandes client
│   ├── reservations.tsx     # Réservations client
│   └── profile.tsx          # Profil + "Voir le menu"
├── (serveur)/
│   ├── _layout.tsx          # Guards + tabs (Commandes, Mon compte)
│   ├── orders.tsx           # Commandes : accepter, préparer, servir
│   └── profile.tsx          # Profil serveur
└── (gerant)/
    ├── _layout.tsx          # Guards + tabs (Produits, Catégories, Utilisateurs, Réservations, Promotions, Stats)
    ├── products.tsx         # CRUD produits
    ├── categories.tsx       # CRUD catégories
    ├── users.tsx            # Gestion utilisateurs + rôles
    ├── reservations.tsx     # Réservations + confirmation
    ├── promotions.tsx       # CRUD promotions
    └── stats.tsx            # Statistiques ventes
```

### API Endpoints (tous préfixés /api)
* Auth: POST /auth/register, POST /auth/login, GET /auth/me
* Products: GET/POST /products, GET/PUT/DELETE /products/:id
* Categories: GET/POST /categories, GET/PUT/DELETE /categories/:id
* Orders: POST /orders (CLIENT), GET /orders, GET /orders/en-attente (SERVEUR), GET /orders/:id, PATCH /orders/:id/statut (SERVEUR/GERANT)
* Reservations: POST /reservations (CLIENT), GET /reservations, PATCH /reservations/:id (GERANT), DELETE /reservations/:id
* Payments: POST /payments (CLIENT), GET /payments (GERANT)
* Reviews: POST /reviews (CLIENT), GET /reviews?produitId=
* Promotions: GET/POST /promotions, GET/PUT/DELETE /promotions/:id
* Users: GET /users (GERANT), PUT/DELETE /users/:id (GERANT)
* Stats: GET /stats (GERANT)

5. FLUX PRINCIPAUX
----------------------------------------------------------

### Commande Client
1. Client parcourt /menu (public)
2. Ajoute au panier → demande connexion si non connecté
3. Va au panier (/client/cart) → modifie quantités +/- 
4. "Passer la commande" → crée Order EN_ATTENTE
5. Serveur voit commande dans "En attente" → "Accepter" → ACCEPTEE
6. Serveur "Préparation" → EN_PREPARATION
7. Serveur "Marquer prête" → PRETE
8. Serveur "Servir" → SERVIE
9. Client paie → PAYEE

### Réservation
1. Client crée réservation → EN_ATTENTE
2. Gérant confirme/annule → CONFIRMEE/ANNULEE

### Promotions
1. Gérant crée promotion (pourcentage/montant, dates)
2. Appliquée côté backend lors du calcul

----------------------------------------------------------
6. SÉCURITÉ & GUARDS
----------------------------------------------------------

* JWT dans Authorization header (Bearer token)
* Roles vérifiés côté backend (preHandler requireRole)
* Côté frontend: layouts (client)/(serveur)/(gerant) redirigent si rôle incorrect
* Token stocké dans AsyncStorage, rechargé au démarrage
* Mot de passe hashé bcrypt (cost 10)
* CORS configuré pour Expo web (localhost:8081) et mobile
